01 · Problem

Feedback quality shouldn't depend on writing skill

Most leaders have observations, but sometimes they struggle to structure them. Raw notes sit in notebooks, inconsistently formatted, and unanchored to any shared framework. The result is feedback that varies in quality, misses development opportunities, and doesn't connect to the competency language that anchors organizational decisions around hiring, calibration, and succession.

The gap

Leaders observe behavior continuously but lack a structured way to translate those observations into actionable, competency-anchored development conversations.

The opportunity

Use AI to meet leaders where they are, accepting raw, unpolished input and returning structured output grounded in the Success Model framework.

02 · Process

Architecting the synthesis engine

The core design challenge was the system prompt, embedding enough competency context that the AI could map unstructured observations to the right framework dimensions without being so prescriptive that it manufactured insights not present in the original input.

1
Framework embedding
The full Stripe Success Model, all 11 competencies, both tiers, and behavioral context, was embedded into the system prompt so every synthesis call is anchored to the same framework, not to general notions of performance.
2
Input design
The input layer was deliberately kept low-friction: name, role, context, and free-form notes. No structured fields that would require leaders to pre-organize their thinking. The tool handles structure; leaders provide observation.
3
Output architecture
Structured JSON output with three categories: Strengths, Opportunities, Recommended Actions. Each item tagged with competency name and tier. The tier label ("The What" / "The How") surfaces the framework logic without requiring the leader to know it in advance.
4
Context signaling
A context selector (Performance review, Project debrief, 360 input, etc.) passes situational framing to the synthesis engine, so the same observations generate appropriately calibrated output depending on where in the talent lifecycle they originate.
5
Interaction design
Side-by-side input/output layout keeps the raw notes visible alongside the synthesis, allowing leaders to verify that the output is grounded in what they actually observed, not invented by the model.

04 · Design decisions

Three choices that shaped the tool

Each decision reflects a deliberate tradeoff between power and usability, and a commitment to keeping the tool grounded in actual observation rather than AI-generated performance language.

01
Framework-first synthesis
Embedding the Success Model in the system prompt — not as a post-processing filter — ensures competency mapping happens during synthesis, not after. Every insight is competency-native.
02
Raw input, not structured input
Forcing leaders to structure their input before submitting would reduce adoption and pre-filter observations. Accepting raw notes removes the friction that keeps feedback unwritten.
03
Tier visibility in output
"The What" / "The How" tier labels surface on every output item, so the framework logic is visible without requiring leaders to study the competency model before using the tool.
05 · Reflection

What this work demonstrates

"The system prompt is the architecture. Getting it right means the tool surfaces genuine insight from rough input rather than generic performance language dressed up in the organization's vocabulary."

This tool demonstrates the ability to move from competency framework design directly into AI-enabled workflow infrastructure, the same motion Stripe is asking for in the Learning Architect role. The Success Model isn't just a static document here. It's the interpretive engine underneath a live tool that managers can use on any given afternoon before a one-on-one.
