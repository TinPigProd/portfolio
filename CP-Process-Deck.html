01 · Brief

The starting point wasn't a deliverable.
It was a question.

"A learning architect role at Stripe requires demonstrating not just what you've built, but how you think. The brief: design a portfolio that shows systems thinking, AI fluency, and design craft, for an audience that will read every word carefully."

The question underneath the brief: How do you make a hiring team feel the quality of your thinking, not just see the output of it?

The answer: build something that demonstrates the skill by using it. A learning architecture portfolio built using learning architecture principles.

02 · Insight

"Don't show four projects.

Building the interpretive engine.

The decision
Korn Ferry has 38 competencies. The question wasn't which ones to include. It was which ones Stripe actually differentiates on.
The answer required reading Stripe's operating principles not as culture statements but as talent signals, each one pointing to a specific behavioral demand.

Why the distinction matters
Many companies value execution.

Fewer explicitly optimize for intellectual humility, curiosity, written rigor, and systems thinking.

Stripe does.

The tier distinction makes that visible, giving managers a framework for calibration conversations that honors both what Stripe requires and what makes it distinctive.

04 · Tool Design

The most important UX decision:
what not to ask for.

Rejected approach
Structured input form
Rate this competency 1–5 ▾
Behavioral example (required)
Select development priority ▾
Submit assessment
Structured input pre-filters observations and reduces adoption. Leaders spend cognitive energy organizing rather than observing.
→
Chosen approach
Open observation field
Name / Role (minimal)
Type raw observations here. Don't worry about structure, write however feels natural...
Synthesize feedback
The tool handles structure. Leaders provide observation. Meet the leader where they are, not where you wish they were.
04 · Tool Design

The system prompt is the architecture.

The insight
In an AI-powered tool, the system prompt is the design document. It determines what the tool can see, how it interprets input, and what kind of output it produces. The framework doesn't filter output after the fact — it shapes the synthesis from the inside.

04 · Tool Design

Bias reduction at the input layer,
not the output layer.

Two options considered
Option A — rejected
Filter outputs after calibration
By the time you filter, the conversation has already happened. The bias has already shaped the manager's thinking.
Option B — chosen
Transform language before placement
The transformation IS the coaching moment. Leaders learn what objective language looks like by seeing the contrast. Bias addressed where it enters, not where it exits.
Before / after example
Raw input — subjective
"He's not the most collaborative though."
↓
Structured output — behavioral evidence
Collaborates · The what
Peers report discovering completed work on items flagged as collaborative, suggesting a default toward independent resolution over shared progress.

05 · Ecosystem Design

The question that designed Beacon.

"When is a manager most motivated to act on a development insight?"

Option A — rejected
Weekly cadence
Monday nudge, regardless of context. Disconnected from the moment. Feels like organizational spam within weeks.
Option B — rejected
Post review cycle
Too infrequent. By the time learning arrives, the insight is months stale and the direct report conversation has moved on.
Option C — chosen
After feedback synthesis
The manager's mind is already on the direct report. The learning arrives when it's most relevant — not when a calendar decides it should be.
The design principle: "Beacon doesn't schedule learning. It detects the moment."
05 · Ecosystem Design

A privacy boundary built into the UX.

The problem
Beacon surfaces in a shared Slack channel (#people-insights). Direct report names shouldn't appear at the channel level — but the learning context needs to be specific enough to be useful.
The solution: two layers of disclosure
Layer 1 — Channel level (visible to all)
"Beacon — based on Direct Report Convo 5.17.26, here are targeted development resources."
Timestamp only. No name. No competency detail.
Layer 2 — Thread level (on deliberate click)
"Direct report: Jordan Mills · Senior Engineer. Flagged: Collaborates · Communicates Effectively"
Name revealed only on intent.
The design principle
"Privacy boundaries should be designed into the interaction model, not bolted on as a compliance afterthought."
What this signals to Stripe
Product design thinking applied to a learning tool. The candidate considers data sensitivity, disclosure design, and enterprise context as part of the UX — not as separate concerns.
